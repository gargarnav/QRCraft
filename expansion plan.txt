SECTION 1 â€” DYNAMIC QR CODES
================================================================================

WHAT IT IS:
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
  A Dynamic QR code looks identical to a static one when printed.
  The difference: instead of encoding the destination URL directly,
  it encodes a SHORT REDIRECT URL that YOU control.

  Example:
    Static QR:  encodes â†’ https://myrestaurant.com/menu-v1.pdf
    Dynamic QR: encodes â†’ https://qrcraft.fun/r/xK9mP2
                          (your redirect URL)
                          which redirects to â†’ https://myrestaurant.com/menu-v1.pdf

  The user can log into QRcraft, edit the destination of xK9mP2,
  and the SAME printed QR code now points somewhere new.
  No reprinting. No new QR code. Magic.

WHY USERS WILL PAY FOR THIS:
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
  - Restaurant updates their menu â†’ no reprint needed
  - Event changes venue â†’ update the QR, same flyers
  - Freelancer updates portfolio â†’ same business card QR
  - This is the #1 upsell feature in the QR code industry

HOW IT WORKS TECHNICALLY:
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
  1. User creates a Dynamic QR code (paid feature)
  2. Backend creates a redirect record:
       id: "xK9mP2"
       destination: "https://user-entered-url.com"
       owner: user_id
       created_at: timestamp
  3. QR code encodes: https://qrcraft.fun/r/xK9mP2
  4. When someone scans it, they hit your server at /r/xK9mP2
  5. Server looks up the destination and does a 302 redirect
  6. User can update the destination anytime from their dashboard

BACKEND NEEDED:
  - A simple redirect endpoint: GET /r/:shortcode â†’ 302 to destination
  - A database table: { shortcode, destination, user_id, created_at }
  - An auth system (simple email/password or Google OAuth)
  - A dashboard page where users manage their dynamic QR codes

RECOMMENDED STACK FOR THIS:
  - Supabase (free tier): handles database + auth + API in one
  - Vercel Edge Functions: handles the /r/:shortcode redirect (fast, free)
  - OR: Keep on Replit and use a simple Express route + MongoDB Atlas (free)

SHORTCODE GENERATION:
  Use nanoid (npm package) to generate 6-character random IDs
  import { nanoid } from 'nanoid'
  const shortcode = nanoid(6) // e.g. "xK9mP2"
  Collision probability at 6 chars (alphanumeric): negligible for MVP scale

REDIRECT PERFORMANCE:
  The redirect must be FAST â€” under 100ms â€” or users will feel lag when scanning.
  Use Vercel Edge Functions (runs at the CDN edge, globally fast) for /r/:shortcode.
  Cache the destination URL in memory for 60 seconds to reduce DB reads.

SECTION 2 â€” SCAN ANALYTICS

WHAT IT IS:
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
  Every time someone scans a Dynamic QR code, QRcraft logs the scan event.
  The owner can view a dashboard showing:
    - Total scan count
    - Scans over time (line chart)
    - Device type breakdown (mobile vs desktop)
    - Browser breakdown (Chrome, Safari, etc.)
    - Country breakdown (where scans happened)
    - Top scanning hours (heatmap)

  This is ONLY available for paid users (Pro+ / $9.99 tier).
  Free users see nothing. Pro ($2.99) sees total scan count only.
  Pro+ ($9.99) sees full analytics.

HOW TO CAPTURE SCAN DATA:
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
  When a user hits GET /r/:shortcode, BEFORE redirecting, log:

  {
    shortcode: "xK9mP2",
    timestamp: new Date().toISOString(),
    user_agent: req.headers['user-agent'],
    ip_country: req.headers['cf-ipcountry'],  // Cloudflare adds this for free
    referrer: req.headers['referer'] || 'direct'
  }

  Parse user_agent with the 'ua-parser-js' npm package:
    - device.type â†’ "mobile" / "tablet" / "desktop"
    - browser.name â†’ "Chrome" / "Safari" / "Firefox"
    - os.name â†’ "iOS" / "Android" / "Windows"

  Store each scan as a row in a 'scans' database table.
  The redirect still happens immediately â€” logging is non-blocking (async).

  Supabase gives you: Postgres DB + Auth + REST API + Edge Functions
  Free tier: 500MB DB, 2 edge functions, 50,000 monthly active users

  Setup (30 minutes):
  1. Create account at supabase.com
  2. Create a new project
  3. Create these tables:

  TABLE: qr_codes
    id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
    user_id     uuid REFERENCES auth.users(id)
    name        text
    shortcode   text UNIQUE
    destination text
    is_dynamic  boolean DEFAULT false
    created_at  timestamptz DEFAULT now()
    updated_at  timestamptz DEFAULT now()

  TABLE: scans
    id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
    shortcode   text REFERENCES qr_codes(shortcode)
    scanned_at  timestamptz DEFAULT now()
    device_type text  -- 'mobile', 'desktop', 'tablet'
    browser     text
    country     text
    os          text

  4. Create an Edge Function for the redirect:
     File: supabase/functions/redirect/index.ts
     Route: /r/:shortcode â†’ lookup â†’ 302 redirect + log scan

  5. Install Supabase client in your project:
     npm install @supabase/supabase-js

  6. Auth: use Supabase's built-in email/password auth
     supabase.auth.signUp({ email, password })
     supabase.auth.signIn({ email, password })

OPTION B â€” REPLIT + EXPRESS + MONGODB ATLAS (Stay on Replit)
