SECTION 1 ‚Äî THE IDEA: OVERVIEW & VISION
================================================================================

NAME IDEAS:
  - QRcraft
  - QRbrush
  - StyleQR
  - QRforge
  - dotQR

ONE-LINER PITCH:
  "Create beautiful, branded QR codes in seconds ‚Äî no design skills needed."

WHAT IT IS:
  A lightweight web app that lets anyone generate fully customized QR codes.
  Unlike free tools that produce plain black-and-white squares, this tool
  allows users to style every aspect of their QR code ‚Äî colors, shapes,
  logo placement, gradients, dot patterns ‚Äî and download a high-quality file
  ready to print or share digitally.

WHY IT'S A GOOD IDEA:
  - QR codes are now everywhere (menus, events, business cards, packaging)
  - Most free tools are ugly and basic
  - Businesses will pay a small fee for something that looks professional
  - Very low competition at the small/micro scale
  - No backend required for MVP ‚Äî runs entirely in the browser
  - Stripe integration takes 30 minutes to set up
  - Repeat traffic is easy with shareability ("look at this cool QR code I made")

VISION:
  Start as a simple tool. Grow into the go-to platform for branded QR codes
  for small businesses, event planners, and content creators.

SECTION 2 ‚Äî PROBLEM & SOLUTION
================================================================================

THE PROBLEM:
  Businesses need QR codes constantly ‚Äî for menus, product packaging, event
  check-ins, business cards, social media, and more. But:

    - Free tools (QR Code Generator, QRStuff) produce generic, ugly codes
    - Design tools (Canva, Figma) require too many steps and skill
    - Professional tools (Beaconstac, QR Tiger) are expensive ($15‚Äì$50/month)
    - Most people don't want a subscription ‚Äî they just want ONE good QR code

THE SOLUTION:
  A dead-simple web app that:
    1. Takes a URL (or text, email, phone number, WiFi credentials, etc.)
    2. Lets you style it with easy visual controls
    3. Shows a live preview that updates instantly
    4. Lets you download in high resolution (PNG, SVG) after a small payment

THE GAP THIS FILLS:
  Between "free and ugly" and "expensive and complex" ‚Äî a simple, beautiful,
  one-time-pay tool for people who care about design but don't want a subscription.

SECTION 5 ‚Äî TECH STACK: FULL BREAKDOWN
================================================================================

PHILOSOPHY:
  Keep it as simple as possible. No database needed for MVP.
  Everything runs in the browser. Payment is handled by Stripe.

------------------------------------------------------------
FRONTEND
------------------------------------------------------------

FRAMEWORK:
  React (with Vite for fast setup)
  OR plain HTML + Vanilla JS (even simpler, no build step)

  Recommendation for speed: Use React with Vite
    - npx create-vite@latest qr-customizer --template react
    - Add Tailwind CSS for styling (fast, no custom CSS needed)

QR CODE LIBRARY:
  qr-code-styling (npm package)
    - Supports: dot shapes, colors, gradients, embedded images
    - Live preview: Yes
    - Output formats: PNG, SVG, JPEG, WEBP
    - GitHub: niclas-timm/qr-code-styling
    - Install: npm install qr-code-styling

  ALTERNATIVE: qrcode.react
    - Simpler, less styling options
    - Good for MVP if qr-code-styling feels complex

COLOR PICKER:
  react-color (npm package)
    - Multiple picker styles: sketch, chrome, swatches
    - Install: npm install react-color

FILE DOWNLOAD:
  Native browser API ‚Äî no library needed
    - For PNG: canvas.toBlob() + URL.createObjectURL()
    - For SVG: serialize the SVG element + Blob download

LOGO UPLOAD:
  Native HTML file input
    - <input type="file" accept="image/*" />
    - FileReader API to convert to base64 and pass to qr-code-styling

PAYMENTS
------------------------------------------------------------

STRIPE CHECKOUT (Recommended):
  - Create a Stripe account (free)
  - Set up a "Payment Link" in the Stripe dashboard (no-code option)
  - OR use Stripe.js for custom integration
  - After successful payment, redirect to download page

  Flow:
    User clicks "Download" ‚Üí Stripe Checkout opens ‚Üí Payment succeeds ‚Üí
    Redirect to /success?session_id=xyz ‚Üí Verify on frontend ‚Üí Enable download

  No backend needed for simple version:
    Use Stripe Payment Links (generates a hosted page ‚Äî zero code)
    After payment, Stripe redirects to your success URL with a token
    Store a "paid" flag in localStorage after redirect

ALTERNATIVE ‚Äî LEMON SQUEEZY:
  - Similar to Stripe but auto-handles EU VAT
  - Great for solo devs who don't want tax headaches
  - Has a simple JS embed

FAQ SECTION CONTENT
------------------------------------------------------------

  Q: Are the QR codes I create actually scannable?
  A: Yes, 100%. All codes are tested to be scannable by any standard camera
     or QR reader app, even with heavy customization. We use proper error
     correction to ensure reliability.

  Q: What formats can I download?
  A: Free users get a low-res PNG. Paid users get high-res PNG (2000x2000px)
     and SVG (infinitely scalable for large print).

  Q: Can I use the QR code commercially?
  A: Yes. Once you download it, it's yours to use however you like ‚Äî
     on products, menus, flyers, packaging, anything.

  Q: Do the QR codes expire?
  A: No. Your QR code is static and works forever as long as the URL
     it points to is active.

  Q: Can I change the URL after downloading?
  A: No ‚Äî the URL is baked into the QR code. If you change your URL,
     you'll need to generate a new code. (Dynamic QR codes are a future feature.)

  Q: I paid but my download didn't work. Help?
  A: Email us at support@[yoursite].com with your receipt and we'll sort it
     out immediately.

  Q: What file size do I get?
  A: High-res PNG files are typically 200KB‚Äì1MB depending on complexity.
     SVG files are under 50KB and scale perfectly to any size.
